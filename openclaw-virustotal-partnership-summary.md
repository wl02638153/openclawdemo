# OpenClaw 與 VirusTotal 合作文章摘要（繁體中文）

## 文章主旨
OpenClaw 宣布與 VirusTotal 合作，將安全掃描導入 ClawHub（OpenClaw 的技能市集）。
所有發佈到 ClawHub 的技能都會經過 VirusTotal 威脅情報與 Code Insight 分析，作為社群安全防線之一。

## 為什麼重要
文章指出，AI Agent 與傳統軟體不同：
- 它會理解自然語言並自行決策行動
- 語言本身可能成為攻擊介面（例如提示注入）
- 技能具備高權限與高影響力，若惡意可造成資料外洩、未授權操作或載入惡意程式

因此，隨著生態擴大，必須把安全機制前置，而不是等事故發生後再補救。

## 技術流程（ClawHub 技能發佈時）
1. **可重現封裝**：將技能檔案打包（含 `meta.json`、版本資訊等）
2. **雜湊計算**：產生整包 SHA-256 指紋
3. **VirusTotal 查詢**：先查既有資料庫與既有分析結果
4. **上傳與分析**：若無結果，透過 VT v3 API 上傳進行新掃描
5. **Code Insight**：使用 LLM（文中提及由 Gemini 驅動）做安全導向程式行為分析
6. **自動審核**：
   - benign：自動核准
   - suspicious：標示警告
   - malicious：立即封鎖下載
7. **每日重掃**：持續檢查既有技能是否轉為惡意

## 能做到什麼、不能做到什麼
### 能提供的價值
- 偵測已知惡意程式與惡意載荷
- 透過行為分析辨識可疑模式
- 提升供應鏈可見度（依賴、嵌入執行檔等）
- 提供使用者判斷信任的額外訊號

### 侷限
- 不是萬靈丹
- 純語言層的惡意誘導或高階提示注入，不一定會被傳統惡意掃描完整捕捉

## 更大安全計畫
OpenClaw 表示這只是第一層，後續還會公布：
- 完整威脅模型
- 公開安全路線圖
- 全程式碼安全稽核細節
- 正式弱點通報流程與 SLA

並提到由 Jamieson O’Reilly 擔任安全顧問，推進整體安全治理。

## 對技能作者的影響
- 新技能會自動進行掃描與分類
- 可在技能頁看到掃描結果與 VT 報告連結
- 若誤判可聯絡 `security@openclaw.ai` 申訴

## 對使用者的建議
即使技能顯示「乾淨」，仍應：
- 檢查技能權限需求
- 優先使用可信任發佈者
- 發現異常即通報

## 一句話總結
這次合作代表 OpenClaw 把「技能供應鏈安全」制度化：以 VirusTotal + Code Insight 作為第一道自動化防線，再配合後續威脅模型與治理機制，朝「可落地、可持續」的 AI Agent 安全體系前進。
